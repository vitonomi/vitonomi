# Test vectors

Golden hex/JSON files referenced from
[../data-format.md](../data-format.md). Implementations MUST round-
trip every vector. CI runs the round-trip on every commit.

## Layout

```
vectors/
├── README.md                     this file
├── chunk/                        encrypted chunk byte vectors
│                                 (delegated — fetched from upstream
│                                 @autonomi/self-encryption release)
├── data-map/                     DataMap byte vectors (delegated to upstream)
├── key-blob/                     key blob envelopes (TBD Phase 2)
├── seed-phrase/                  seed → keypair derivation (TBD Phase 2)
├── snapshot/                     snapshot envelope vectors (TBD Phase 5)
├── head-pointer/                 head pointer envelopes (TBD Phase 5)
├── record-frame/                 per-record-type frames (TBD Phase 5)
├── alias-pubkey/                 alias-pubkey directory entries (TBD Phase 8)
└── admin-chain/                  admin chain entries (TBD Phase 9)
```

Each subdirectory has its own README explaining the format
generator, the input domain covered, and any known edge cases.

## Regeneration policy

Vectors are regenerated by `scripts/regen-vectors.sh` (TBD —
created in Phase 2 alongside the first vector set). Regenerated
vectors are committed alongside the spec change that produced them.

## Checksum policy

Each subdirectory contains a `MANIFEST.txt` with one line per
vector: `<sha256-hash>  <filename>`. CI verifies that every vector
file matches its manifest entry. A mismatch fails the build.
